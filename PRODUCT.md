## 核心玩法规则

### 采集机制

- **每日灵感采集倍数**：
  - 第 1 条：1.0 倍
  - 第 2 条：1.5 倍
  - 第 3 条：2.0 倍
  - 第 4 条及以后：0.2 倍

- **基础星点奖励**：10 星点（可在 `config.yaml` 中配置）
- **提交要求**：必须添加至少一个标签

### 任务与奖励

| 任务类型 | 奖励星点 | 触发时间 | 扫描范围 | 说明 |
|---------|---------|---------|---------|------|
| 周回顾 | 200 | 次周任意时间（手动点击） | 上一自然周（周一→周日） | 系统生成上周碎片素材文件，用户编辑后提交 |
| 月度战报 | 500 | 次月任意时间（手动点击） | 上一自然月（1日→月末） | 系统生成上月碎片素材文件，用户编辑后提交 |
| 内容输出（媒体发布） | 750-1000 | 随时 | — | 每次发布，最多 5 次/轮 |

### 回顾与战报流程

**周回顾流程**：
1. 用户在次周点击 [📅 周回顾]
2. 后端自动扫描上一自然周（周一→周日）的所有灵感笔记
3. 生成「周回顾素材-YYYY年第X周.md」文件，存入 Inbox/
4. 后端返回文件链接和内容预览给前端
5. 用户在前端查看素材内容，在输入框中撰写本周核心洞察
6. 点击提交 → 后端将用户总结写入素材文件，自动提取素材中所有标签，在文件顶部添加 frontmatter（date/tags/hunt），标签合并 `#周回顾`
7. 系统发放 200 星点，更新完成状态

**月度战报流程**：
1. 用户在次月点击 [📈 月度战报]
2. 后端自动扫描上一自然月（1日→月末）的所有灵感笔记
3. 生成「月度战报素材-YYYY年X月.md」文件，存入 Inbox/
4. 后端返回文件链接和内容预览给前端
5. 用户在前端查看素材内容，在输入框中撰写本月核心洞察
6. 点击提交 → 后端将用户总结写入素材文件，自动提取素材中所有标签，在文件顶部添加 frontmatter（date/tags/hunt），标签合并 `#月回顾`
7. 系统发放 500 星点，更新完成状态

**素材文件前端展示规范**：

回顾弹窗中展示素材内容时，需对内容做以下处理：

1. **过滤 frontmatter**：不显示 YAML 元数据（`---` 包裹的部分），只展示正文。
2. **信息层级**：
   - 一级标题（`#`）：使用大号字体（1.4rem）、加粗、深色
   - 二级标题（`##`）：使用中号字体（1.1rem）、加粗、主题色
   - 笔记标题：使用加粗样式，与摘要区分
   - 正文/摘要：使用常规字体（0.9rem）、常规颜色
   - 日期和标签：使用小号字体（0.75rem）、灰色、放置在笔记标题下方作为辅助信息
3. **排版**：
   - 每条笔记之间用浅色分隔线隔开
   - 笔记标题、辅助信息、正文之间保持合理的行间距
   - 表格仅用于后端生成文件，前端展示时转为列表式排版
4. **复制按钮**：弹窗顶部提供"复制全文"按钮，点击后将素材文件完整内容（含 frontmatter）复制到剪贴板。
5. **标签渲染**：标题行中的 `#tag` 自动拆出，以不同灰色渲染（视觉层级最低）。
6. **素材分隔**：素材之间使用渐变线 + 居中菱形 `◆` 装饰分隔。
7. **视觉层级**：标题（主题色渐变底纹+左边框）> 日期（主题色浅底纹）> 标签（灰色系）

### 勋章系统

| 勋章名称 | 获得条件 |
|---------|---------|
| 灵光乍现 | 首次单日完成 3 条灵感采集 |
| 周常猎人 | 连续 4 周完成每周回顾 |
| 连线大师 | 单篇月度战报中连接 3 个以上不同领域碎片 |

### 兑换系统

- **消费券兑换**：1 星点 = 1 元
- **基金充值**：1 星点 = 基础汇率 × (1 + 基金加成%)
  - 基金加成由月度勋章数量决定
  - 最高加成：25%

### 惩罚机制

- **连续采集中断**：收益减半
- **惩罚状态**：影响所有采集收益
- **恢复机制**：完成指定任务可解除惩罚

## 技术架构

### 前端技术栈

- **框架**: Alpine.js 3.x
- **语言**: Vanilla JavaScript
- **样式**: 纯 CSS（内联样式 + 全局样式）
- **图标**: Emoji 表情符号
- **特点**: 单页应用，无需构建工具，开箱即用

### 后端技术栈

- **框架**: FastAPI 0.104.x
- **语言**: Python 3.10+
- **依赖**:
  - Uvicorn（ASGI 服务器）
  - Pydantic（数据验证）
  - PyYAML（配置文件）

### 项目文档体系

| 文档 | 定位 | 更新原则 |
|------|------|----------|
| `README.md` | 项目概览与快速入门 | 重大变更时更新 |
| `PRODUCT.md`（本文档） | 产品规格说明书——完整设计规则与技术实现细节 | 始终反映最新状态，不记录历史 |
| `LOGS.md` | 开发日志与任务追踪——当前任务清单 + 每日决策/问题/计划 | 每次开发会话后追加 |

> **关键约定**：PRODUCT 只描述「现在是什么」，LOGS 记录「怎么变成这样的」。不要在 PRODUCT 中写变更日志。

### LOGS.md 撰写规范

- **当前任务区**：用 checkbox 管理待办/进行中/已完成，新任务加在最前面
- **详细日志区**：按日期倒序，每条记录包含：做了什么、决策理由、遇到的问题、下一步计划
- **更新时机**：切换设备前、完成子任务时、每天结束时

### 程序文件说明

| 文件 | 职责 |
|------|------|
| `main.py` | FastAPI 路由层，对接 HTTP 请求到 engine，提供所有 `/api/*` 端点 |
| `engine.py` | 核心游戏逻辑：星点计算、状态管理、灵感采集、回顾/战报素材生成、勋章判定、兑换处理 |
| `static/index.html` | 单页前端（Alpine.js），包含全部 UI 与交互逻辑，无构建工具，开箱即用 |
| `data/inspire/_狩猎系统/state.json` | 游戏状态持久化文件（星点余额、连续天数、勋章、回顾/战报完成状态等） |
| `data/inspire/_狩猎系统/config.yaml` | 系统配置参数（基础星点、倍数、奖励金额、汇率等） |
| `requirements.txt` | Python 依赖声明 |
| `.gitignore` | 忽略规则（venv/、__pycache__/、.DS_Store 等） |
| `LOGS.md` | 开发日志与任务追踪，记录当前任务清单和每日决策/问题/计划 |

## API 接口

### 接口列表

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | `/api/capture` | 提交灵感采集 |
| GET | `/api/status` | 获取游戏状态 |
| POST | `/api/reset` | 重置游戏状态（开发/测试用） |
| POST | `/api/content/verify` | 提交内容输出链接并验证 |
| POST | `/api/exchange` | 兑换星点 |
| POST | `/api/review/weekly` | 生成上周素材文件，返回文件路径和内容预览 |
| POST | `/api/review/weekly/submit` | 提交周回顾总结，写入素材文件，添加 frontmatter + `#周回顾` 标签，发放星点 |
| POST | `/api/report/monthly` | 生成上月素材文件，返回文件路径和内容预览 |
| POST | `/api/report/monthly/submit` | 提交月度战报总结，写入素材文件，添加 frontmatter + `#月回顾` 标签，发放星点 |
| GET | `/api/report/monthly/draft` | 获取月度战报草稿 |
| GET | `/api/config` | 获取配置 |
| PUT | `/api/config` | 更新配置 |

### 请求/响应示例

#### POST /api/capture

**请求**：
```json
{
  "content": "灵感内容...",
  "tags": ["标签1", "标签2"],
  "folder": "Inbox"
}
```

**响应**：
```json
{
  "success": true,
  "message": "捕获成功！",
  "star_earned": 15.0,
  "total_star": 100.0
}
```

#### POST /api/exchange

**请求**：
```json
{
  "type": "coupon",
  "amount": 50
}
```

**响应**：
```json
{
  "success": true,
  "message": "兑换成功！",
  "deducted_star": 50,
  "real_value": 50.0
}
```

## 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `base_star` | 10 | 基础星点奖励 |
| `daily_multipliers` | [1.0, 1.5, 2.0, 0.2] | 每日采集倍数 |
| `weekly_review_reward` | 200 | 周回顾奖励 |
| `monthly_report_reward` | 500 | 月度战报奖励 |
| `content_output_reward_min` | 750 | 内容输出最低奖励 |
| `content_output_reward_max` | 1000 | 内容输出最高奖励 |
| `content_output_max_times` | 5 | 每轮最多发布次数 |
| `coupon_rate` | 1.0 | 消费券汇率 |
| `fund_base_rate` | 1.5 | 基金基础汇率 |
| `custom_fund_name` | "我的投资账户" | 基金名称 |

## 部署方式

### 开发模式

```bash
python3 -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

### 生产模式（Mac Mini）

1. 在 Mac mini 上创建项目目录并安装依赖
2. 配置 launchd 实现开机自启
3. 安装 Tailscale 并在所有设备登录同一账号
4. 其他设备通过 Tailscale 分配的 IP + 端口访问
5. 如需公网访问，配置 Cloudflare Tunnel

### 网络架构

```
用户设备
    ↓
Tailscale 虚拟网络
    ↓
Mac Mini (FastAPI 服务)
    ↓
iCloud Drive (Obsidian Vault)
```

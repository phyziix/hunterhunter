#工作流 #跨设备同步 #Git #LOGS方案 #效率工具

## 背景与目标

- 设备：Mac mini（家里常开）、MacBook Air（公司主力）、iPhone/iPad（移动查看）
- 核心工具：
  - 腾讯 WorkBuddy、QClaw（作为调度接口，目前储备）
  - 字节 TRAE IDE、TRAE Solo、腾讯 CodeBuddy（编码工具）
- 目标：实现单项目在多设备、多 IDE 之间的跨设备接续开发，不依赖工具私有对话历史，避免信息丢失。

## 核心问题

- 同一时间只能在一个设备上使用一个 IDE（单线程操作）
- TRAE IDE、CodeBuddy、TRAE Solo 的对话历史不跨设备
- WorkBuddy 与 QClaw 目前未承担具体任务，仅作为储备

## 最终方案：三文件同步 + 手动接续

放弃复杂自动化，采用极简的以文件为中心的手动接续模式。

### 文件结构（项目根目录）

    your-project/
    ├── README.md      # 项目说明（极少改动）
    ├── PRODUCT.md     # 产品功能最终态（完成功能时更新）
    └── LOGS.md        # 开发日志 + 任务清单（核心）

### LOGS.md 模板

    # 开发日志与任务

    ## 🔥 当前任务（最新）
    - [ ] 任务3：响应式适配按钮
    - [x] 任务2：修改按钮颜色为蓝色
    - [x] 任务1：添加欢迎横幅

    ## 📝 详细日志（最新的在上）

    ### 2026-05-21 16:00 @ MacBook Air / TRAE IDE
    - 做了什么：
    - 决策：
    - 遇到问题：
    - 下一步：
    - 备注：

### 工作流

1. 在设备 A 打开项目 → git pull（或 iCloud 自动同步）→ 阅读 LOGS.md 顶部 → 知道下一步
2. 编码/修改 → 完成后更新 LOGS.md（追加日志、勾选任务）
3. git add . && git commit -m "更新日志" && git push（或用 TRAE Solo 扫码推送）
4. 换到设备 B → git pull → 阅读更新后的 LOGS.md → 无缝接续

### 关键原则

- 不依赖任何 IDE 的对话历史
- 所有跨设备信息流转只依赖 LOGS.md
- 手动维护，简单可靠
- 产品需求和最终状态记录在 PRODUCT.md，与开发日志分离

## 拓扑图（纯文本）

    ☁️ 文件同步层：iCloud Drive 或 Git 远程仓库
        │
        ├── 🏠 Mac mini (TRAE IDE, CodeBuddy, TRAE Solo, Git)
        ├── 💻 MacBook Air (TRAE IDE, CodeBuddy, TRAE Solo, Git)
        └── 📱 iPhone/iPad (TRAE Solo Web 或 微信阅读)

    项目根目录（同步）：
        ├── README.md
        ├── PRODUCT.md
        └── LOGS.md ◄── 核心

    工作流：
        设备A：git pull → 读 LOGS.md → 工作 → 更新 LOGS.md → git push
        设备B：git pull → 读 LOGS.md → 继续 → 更新 LOGS.md → git push

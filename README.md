# 认知狩猎 (Cognitive Hunt) 

## 1. 项目概述 

### 一句话描述
一款基于灵感采集的游戏化知识管理工具，通过捕捉日常灵感积累星点，兑换奖励，培养持续学习习惯。

### 核心价值
- 将零散灵感转化为可量化的知识资产
- 游戏化机制激励持续创作和学习
- 建立个人知识体系的闭环管理

### 目标用户
- 创作者、写作者、研究者
- 希望培养每日写作习惯的人群
- 注重知识积累和整理的知识工作者

## 2. 技术架构 

### 前端
- **技术栈**: Alpine.js 3.x + Vanilla JavaScript
- **样式**: 纯 CSS (内联样式 + 全局样式)
- **图标**: Emoji 表情符号
- **特点**: 单页应用，无需构建工具，开箱即用

### 后端
- **框架**: FastAPI 0.104.x
- **语言**: Python 3.10+
- **依赖**: Uvicorn (ASGI 服务器), Pydantic (数据验证), PyYAML (配置文件)

### 数据存储
- **开发阶段**: `data/inspire/` 目录（项目内）
- **正式上线**: iCloud Drive 中的 Obsidian Vault  
  路径: `~/Library/Mobile Documents/iCloud~com~obsidianmd~Obsidian/Documents/inspire`
- **灵感笔记**: Markdown 格式，存储于 `Inbox/` 文件夹
- **游戏状态**: JSON 格式 (`_狩猎系统/state.json`)
- **配置参数**: YAML 格式 (`_狩猎系统/config.yaml`)
- **兑换记录**: Markdown 格式 (`_狩猎系统/exchange_log.md`)

### 部署方式
- **开发模式**: `python3 -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload`
- **生产模式**: Mac mini 上运行 Python FastAPI，通过 Tailscale 虚拟网络供其他设备访问
- **公网访问**: 如需暴露到公网，使用 Cloudflare Tunnel

## 3. 核心玩法规则 

### 采集机制
- 每日灵感采集有次数加成：第1条1倍、第2条1.5倍、第3条2倍、第4条及以后0.2倍
- 基础星点可配置（默认10星点）
- 必须添加至少一个标签才能提交

### 任务与奖励
- **周回顾**: 完成每周回顾获得 200 星点
- **月度战报**: 完成月度报告获得 500 星点
- **内容输出（媒体发布）**: 每次发布获得 750-1000 星点，最多 5 次/轮

### 勋章系统
- **灵光乍现**: 首次单日完成 3 条灵感采集
- **周常猎人**: 连续 4 周完成每周回顾
- **连线大师**: 单篇月度战报中连接 3 个以上不同领域碎片

### 兑换系统
- **消费券兑换**: 1星点 = 1元
- **基金充值**: 1星点 = 基础汇率 × (1 + 基金加成%)
- 基金加成由月度勋章数量决定（最高25%）

### 惩罚机制
- 连续采集中断后收益减半
- 惩罚状态会影响所有采集收益

```json
{

  "penalty_active": false,
  "penalty_days": 0,
  "medals": [],
  "monthly_medals": 0,
  "gray_medal": false,
  "last_weekly_review": "",
  "last_monthly_report": "",
  "weekly_review_done": false,
  "monthly_report_done": false,
  "exchange_history": [],
  "published_count": 0

```

### 配置参数列表
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `base_star` | 10 | 基础星点奖励 |
| `daily_multipliers` | [1.0, 1.5, 2.0, 0.2] | 每日采集倍数 |
| `weekly_review_reward` | 200 | 周回顾奖励 |
| `monthly_report_reward` | 500 | 月度战报奖励 |
| `content_output_reward_min` | 750 | 内容输出最低奖励 |
| `content_output_reward_max` | 1000 | 内容输出最高奖励 |
| `content_output_max_times` | 5 | 每轮最多发布次数 |
| `coupon_rate` | 1.0 | 消费券汇率 |
| `fund_base_rate` | 1.5 | 基金基础汇率 |
| `custom_fund_name` | "我的投资账户" | 基金名称 |

## 5. API 接口 

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | `/api/capture` | 提交灵感采集 |
| GET | `/api/status` | 获取游戏状态 |
| POST | `/api/content/verify` | 提交内容输出链接并验证 |
| POST | `/api/exchange` | 兑换星点 |
| POST | `/api/review/weekly` | 完成周回顾 |
| POST | `/api/report/monthly` | 完成月度战报 |
| GET | `/api/report/monthly/draft` | 获取月度战报草稿 |
| GET | `/api/config` | 获取配置 |
| PUT | `/api/config` | 更新配置 |

### 请求/响应示例

**POST /api/capture**
```json
// 请求
{
  "content": "灵感内容...",
  "tags": ["标签1", "标签2"],
  "folder": "Inbox"
}

// 响应
{
  "success": true,
  "message": "捕获成功！",
  "star_earned": 15.0,
  "total_star": 100.0
}
```

**POST /api/exchange**
```json
// 请求
{
  "type": "coupon",
  "amount": 50
}

// 响应
{
  "success": true,
  "message": "兑换成功！",
  "deducted_star": 50,
  "real_value": 50.0
}
```

## 7. 开发环境与部署 

### 环境依赖
```bash
fastapi==0.104.1
uvicorn==0.24.0
pydantic==2.5.2
pyyaml==6.0.1
```

### 启动命令
```bash

pip install -r requirements.txt

# 开发模式（自动重载）
python3 -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload

uvicorn main:app --host 0.0.0.0 --port 8000
```

### 部署步骤
1. 在 Mac mini 上创建项目目录并安装依赖
2. 配置 launchd 实现开机自启
3. 安装 Tailscale 并在所有设备登录同一账号
4. 其他设备通过 Tailscale 分配的 IP + 端口访问
5. 如需公网访问，配置 Cloudflare Tunnel

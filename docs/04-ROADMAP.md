## Phase 2: 能力值系统 + 勋章系统独立

**阶段目标**：实现采集力、连接力、输出力三个能力值的计算和展示；将勋章系统从引擎代码中抽离，改为读取 `config.yaml` 的 `medals` 配置表。

**依赖**：Phase 1（数据结构迁移完成）

**预计工期**：3-4 天

### 任务清单

- [ ] **采集力计算**
  - [ ] 在 `engine.py` 中实现 `_calculate_hunt_power()` 函数
  - [ ] 公式：`(total_notes^0.7) × log(active_days+1) × (1 + streak_days/100)`
  - [ ] 范围：5~200
  - [ ] 在每次采集后更新采集力

- [ ] **连接力计算**
  - [ ] 在 `engine.py` 中实现 `_calculate_link_power()` 函数
  - [ ] 公式：`密度 × 100 × log₂(节点数+1) × (跨界占比+0.1)`
  - [ ] 范围：0.1~50
  - [ ] 在每次采集后更新连接力

- [ ] **输出力计算**
  - [ ] 在 `engine.py` 中实现 `_calculate_output_power()` 函数
  - [ ] 公式：`(published_count^0.6) × (1 + reports/10) × log(输出总星点+1)`
  - [ ] 范围：0~100
  - [ ] 在内容输出和战报提交后更新输出力

- [ ] **能力值变更日志**
  - [ ] 实现 `_update_ability_changes(ability, change, reason)` 函数
  - [ ] 记录最近7天的变更（超过7天的自动清理）
  - [ ] 在前端能力面板展示变更摘要

- [ ] **前端能力面板**
  - [ ] 在 `static/index.html` 中添加能力值展示区域
  - [ ] 三个进度条分别显示采集力/连接力/输出力
  - [ ] 展示最近变更日志（最多7条）
  - [ ] 显示解锁的称号和效果

- [ ] **能力值解锁逻辑**
  - [ ] 采集力 ≥ 30：解锁称号「采集达人」🏹
  - [ ] 采集力 ≥ 100：第4条采集倍数从 0.2 提升至 0.4
  - [ ] 连接力 ≥ 5：解锁称号「连线新手」🕸️
  - [ ] 连接力 ≥ 20：基金加成额外 +5%
  - [ ] 输出力 ≥ 10：解锁「内容投资券」（高汇率，锁定30天）
  - [ ] 输出力 ≥ 50：内容输出奖励下限提升至 850 星点

- [ ] **勋章系统独立（从引擎抽离）**
  - [ ] 实现 `check_medals()` 函数，读取 `config.yaml` 的 `medals` 配置表
  - [ ] 原能力值解锁任务改为调用 `check_medals()`
  - [ ] 实现 `get_all_medals()` 函数，返回所有勋章定义（含已获得/未获得状态）
  - [ ] 实现 `get_medal_status(medal_id)` 函数，检查勋章是否已获得
  - [ ] 勋章触发时机：
    - 采集成功后检查 event 类型勋章（如 `daily_capture_count`）
    - 能力值更新后检查 ability 类型勋章（如 `hunt_power >= 30`）
    - 赛季结算时检查 season 类型勋章
  - [ ] `once: true` 的勋章只能获得一次
  - [ ] 前端勋章墙从 `get_all_medals()` 获取勋章列表

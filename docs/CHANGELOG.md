## v0.2.4（2026-05-23）🔧 开发中

## v0.2.3（2026-05-21）

### 星点里程碑系统重新设计

**关键决策**：废除 v0.22 的独立能力值计算体系(采集力/输出力)，改为累计星点档位制(500/1000/2000/3000)。连接力保留独立计算，作为后台引擎触发一次性星点奖励。

- 删除 `_calculate_hunt_power()` 和 `_calculate_output_power()`
- 新增跨界采集加成(≥2个不同领域标签，+5星点)
- 新增赛季星点上限 3000，结算软重置保留 50%
- 移除 `ability_changes` 日志(仅保留连接力)
- 移除 `hunt_apprentice` 勋章(由星点里程碑"初露锋芒"替代)

### 连接力完善

公式：`图密度 × 100 × log₂(节点数+1) × (跨界占比+0.1)`，范围 0.1~50。每个阈值每赛季只触发一次。前端不展示数值，仅在触发时 Toast 提示。

## v0.2.2（2026-05-20）

### 核心架构变更

**关键决策**：废除惩罚机制，改为连续天数递增奖励。这不是"不做就惩罚"，而是"做了就有奖励"。基金池与可用星点分离存储。

- Phase 1 完成：state.json 从 v0.1 迁移到 v0.22，新增 15+ 字段
- 基金池分离：`available_star`(消费券) / `fund_pool`(基金)
- 连续加成：7天+10% / 30天+30% / 90天+50%
- 连续选择奖惩：基金连续2/4/8周 1.55x→1.6x→1.65x；消费券 0.95x→0.9x→0.85x
- 镜像对比：兑换确认页强制展示三层(等值对比/时间维度/累计损失)
- 标签共现图(tag_graph)初始化
- defaults.yaml 初始值模板

### 新增系统

- **能力值系统**：采集力、连接力、输出力三维度(v0.23 重新设计)
- **标签宇宙**：标签云、排行榜、标签详情弹窗+关联笔记
- **赛季系统**：90天赛季→v0.2.4改为15天、主题(开拓者/连线大师/深度矿工/分享者)
- **三层反馈**：微观(采集瞬间)、中观(能力面板+周期回顾)、宏观(长期推演+赛季结算)

### Bug 修复

标签必填校验、基金30天锁定期、镜像对比弹窗完善、勋章同步、Alpine.js 兼容性修复等。

## v0.2.4（2026-05-23~24）✅ 已上线

### 基金池系统重构 + 动态锁定（05-23）

**关键决策**：基金从"30天固定锁定"改为"每日一次动态锁定(24时重置)"。基础倍率从 1.5 改为 1.0 + 常设基础奖励 0.5。所有增益减益改为**加减关系**。

- 新增 `coupon_pool` 记录消费兑换累计，`last_exchange_date` 用于锁定校验
- 基金最低提取额度 500 星点
- 前端水位进度条基于上限 5000 星点

### 正文标签提取

textarea 与标签输入框之间新增"提取正文中的 #标签"按钮，自动识别 `#标签名` 格式，去重合并。

### 获取记录功能

新增 `star_income_history` 字段 + `/api/income/history` API + 前端弹窗。记录来源：灵感采集、连接力奖励、内容发布、周回顾、月度战报。

### Bug 修复

- Alpine.js 语法混用(`{{ }}`→`x-text`)

- 标签详情弹窗乱码
- 获取记录按钮无反应
- 移动端适配缺失（零 `@media` 查询，兑换中心双列按钮窄屏溢出）
- 默认配置 fund.base_rate 1.5→1.0、path_bonuses 绝对值→增量值（对齐 config.yaml）
- 删除 engine.py 重复 _check_season_medal 方法
- process_daily_capture 改用 _calculate_stars()（daily_multipliers 倍增阶梯此前未生效）
- index.html config.fund 初始值缺失（Alpine 初始化报错 `min_withdraw` undefined）
- docs/STATUS.md 调试规则补运行时启动浏览器 + 清理步骤

- **iCloud 同步缺失**：v0.1→v0.2.4 升级漏掉 `engine.backup()`/`sync_to_icloud()`/`start_icloud_sync()` 三个方法、三个对应 API 端点、`sync_to_icloud.py`（根因：无特征连续性检查）→ 05-24 补回并验证

### 特征连续性补回（05-24）

v0.1→v0.2.4 升级时遗漏的 iCloud 同步功能全部补回：
- engine.py 补回 `backup()` / `sync_to_icloud()` / `start_icloud_sync()` 三个方法
- main.py 补回 `/api/backup` / `/api/backup/list` / `/api/sync/icloud` / `/api/reset/daily` 端点
- 新增 `sync_to_icloud.py` 独立同步脚本

### 相似笔记算法升级（05-24）

`_find_related_notes()` 从简单标签 Jaccard 改为 IDF 加权 Jaccard：
- 高频标签降权（如 #技术 出现 50 次权重低），低频标签升权（#冷门领域 出现 1 次权重高）
- 内容相似度权重从 0.3 提升到 0.5
- 阈值收紧：必须标签重叠 ≥2 且综合得分 ≥0.2

### 规则与文档完善（05-24）

**安全防护**：
- iCloud 操作权限：对 `~/Library/Mobile Documents/` 和 `<PRODUCTION>/` 的任何写删操作必须先获用户同意
- WF 自保护修改权限：AI 不得自行修改 `AI_WORKFLOW.md`，必须先获用户明确同意
- 调试规则补强：删数据前必须先备份、恢复时优先查 `~/Documents/hunterhunter_backups/`、禁止在 workspace 调试端测试 sync/backup

**部署规则细化**：
- 备份步骤明确化：对象（生产 `data/`）、原则（只增不替）、命令（`cp -r`）
- 新增特征连续性检查步骤（v0.1→v0.2.4 漏 iCloud sync 为触发案例）

### Tunnel 503 自动恢复（05-24）

tunnel.sh 从一次性启动改为持续守护循环：
- 运行中每 10s 检测后端健康，后端挂了自动 kill lt 并重连
- 部署规则第 9 步改为每次部署必须重启隧道（之前只在端口变化时重启）
- 根因：停 FastAPI 时 localtunnel 不崩溃只显 503，launchd 不会自动重启

## v0.2.3（2026-05-21）

### 星点里程碑系统重新设计

**关键决策**：废除 v0.22 的独立能力值计算体系(采集力/输出力)，改为累计星点档位制(500/1000/2000/3000)。连接力保留独立计算，作为后台引擎触发一次性星点奖励。

- 删除 `_calculate_hunt_power()` 和 `_calculate_output_power()`
- 新增跨界采集加成(≥2个不同领域标签，+5星点)
- 新增赛季星点上限 3000，结算软重置保留 50%
- 移除 `ability_changes` 日志(仅保留连接力)
- 移除 `hunt_apprentice` 勋章(由星点里程碑"初露锋芒"替代)

### 连接力完善

公式：`图密度 × 100 × log₂(节点数+1) × (跨界占比+0.1)`，范围 0.1~50。每个阈值每赛季只触发一次。前端不展示数值，仅在触发时 Toast 提示。

## v0.2.2（2026-05-20）

### 核心架构变更

**关键决策**：废除惩罚机制，改为连续天数递增奖励。这不是"不做就惩罚"，而是"做了就有奖励"。基金池与可用星点分离存储。

- Phase 1 完成：state.json 从 v0.1 迁移到 v0.22，新增 15+ 字段
- 基金池分离：`available_star`(消费券) / `fund_pool`(基金)
- 连续加成：7天+10% / 30天+30% / 90天+50%
- 连续选择奖惩：基金连续2/4/8周 1.55x→1.6x→1.65x；消费券 0.95x→0.9x→0.85x
- 镜像对比：兑换确认页强制展示三层(等值对比/时间维度/累计损失)
- 标签共现图(tag_graph)初始化
- defaults.yaml 初始值模板

### 新增系统

- **能力值系统**：采集力、连接力、输出力三维度(v0.23 重新设计)
- **标签宇宙**：标签云、排行榜、标签详情弹窗+关联笔记
- **赛季系统**：90天赛季→v0.2.4改为15天、主题(开拓者/连线大师/深度矿工/分享者)
- **三层反馈**：微观(采集瞬间)、中观(能力面板+周期回顾)、宏观(长期推演+赛季结算)

### Bug 修复

标签必填校验、基金30天锁定期、镜像对比弹窗完善、勋章同步、Alpine.js 兼容性修复等。

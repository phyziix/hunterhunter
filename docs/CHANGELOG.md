- **惩罚代码清理**：删除前端 penalty 相关死代码（v0.2.2 已废除），涉及 index.html（`:class` 绑定 + 提示条）、styles.css（`.penalty-indicator` + `@keyframes pulse`）、app.js（`penalty_active` 初始值）

## v0.3.2（2026-05-24）✅ 已上线

### 后端拆分

将单文件 `engine.py`（2258行）按功能拆分为多个模块，采用 **Mixin 多重继承** 模式：

| 模块 | 文件 | 行数 | 功能 |
|------|------|:----:|------|
| Core | `engine_core.py` | ~500 | 状态/配置/日志/数据路径 |
| Backup | `engine_backup.py` | ~80 | 备份/iCloud 同步 |
| Season | `engine_season.py` | ~200 | 赛季/主题/结算 |
| Review | `engine_review.py` | ~350 | 周回顾/月战报 |
| Exchange | `engine_exchange.py` | ~200 | 兑换/基金/汇率 |
| Capture | `engine_capture.py` | ~800 | 采集/标签/发现/连接力 |

`engine.py` 最终简化为仅 import + class 定义（约50行）。

### 设计决策

- **Mixin 模式**：API 层零变更，`engine.xxx()` 调用不受影响
- **执行顺序**：Core → Backup → Season → Review → Exchange → Capture（依赖最少的先拆）
- **每拆一模块立即验证**：精准定位问题

优化标签提取正则，避免误匹配 `## 标题` 等 Markdown 格式：
- 正则从 `#[^\s.,;:!?，。；：！？、\n]+` 改为 `#[^\s.,;:!?，。；：！？、\n#][^\s.,;:!?，。；：！？、\n]*`
- 确保 `#` 后面直接跟有效字符，过滤掉 `##`、`###` 等情况

### 相关灵感匹配规则再优化

针对 IDF 加权 Jaccard 匹配的问题进行系统性调整：
- **IDF 上限截断**：将 IDF 最大值限制在 5.0，防止冷门标签（全库只出现1次）权重爆炸
- **标签/内容权重调整**：从 5:5 改为 7:3，更重视用户主动打的结构化标签
- **排序键调整**：从 `(标签重叠数 DESC, 综合得分 DESC)` 改为 `(综合得分 DESC, 标签重叠数 DESC)`
- **门槛放宽**：保留原规则（标签重叠≥2 且 综合得分≥0.2），新增「标签重叠=1 且 综合得分≥0.25」的规则，支持单标签新笔记

### 标签宇宙优化

对标签宇宙模块进行全面优化：
- **字号映射优化**：从线性映射改为对数映射，让高频和低频标签的大小差距更温和
- **加载状态与错误提示**：标签详情弹窗新增加载动画和错误提示，支持重试
- **性能优化**：新增 `tag_index` 索引，按标签查询笔记从 O(n) 优化到 O(1)
- **视觉优化（团队建议）**：
  - 降低基准字号（14px→11px），限制最大字号（20px）
  - 减小内边距（8px 14px→2px 8px）、减小间距（10px→4px）、减小行间距
  - 增加长标签截断（max-width 150px + 省略号）
- **低频过滤**：标签云和排行榜只显示使用次数 ≥2 的标签，避免噪音
- **排行榜限制**：只显示前 10 个高频标签
- **动态高度**：标签云容器高度由内容撑起，不再固定高度

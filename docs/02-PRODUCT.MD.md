## 一、核心玩法规则

### 1.1 灵感采集

- **每日采集倍数**：第 1 条 1.0x / 第 2 条 1.5x / 第 3 条 2.0x / 第 4 条及以后 0.2x
- **基础星点**：10 星点（可在 `config.yaml` 中配置）
- **提交要求**：必须添加至少一个标签
- **标签规则：**
	- 标签提交不限制数量
	- 连接力计算只取前 5 个（两两配对、更新 `tag_graph`）
	- 全部标签写入笔记 frontmatter，用于检索和标签宇宙展示
	- 跨界采集加成：单篇笔记使用 ≥2 个不同领域标签时，额外 +5 星点
	- 标签由 AI 生成、用户粘贴（保证公正性——AI 不知道连接力规则，用户无法刻意操纵跨界组合）


### 1.2 任务与奖励

| 任务类型 | 奖励星点 | 触发时间 | 说明 |
|---------|---------|---------|------|
| 周回顾 | 200 | 次周任意时间 | 系统生成素材文件，用户编辑后提交 |
| 月度战报 | 500 | 次月任意时间 | 系统生成素材文件，用户编辑后提交 |
| 内容输出（媒体发布） | 750-1000 | 随时 | 每次发布，最多 5 次/轮 |

### 1.3 回顾与战报流程

1. 用户在次周/次月点击触发按钮
2. 后端自动扫描上一周期所有灵感笔记
3. 生成素材文件存入 Inbox/
4. 返回文件链接和内容预览给前端
5. 用户撰写核心洞察并提交
6. 后端写入用户总结，自动提取所有标签，添加 frontmatter（date/tags/hunt），合并 `#周回顾` 或 `#月回顾`
7. 发放星点，更新完成状态

### 1.4 素材文件前端展示规范

- 过滤 frontmatter，只展示正文
- 视觉层级：标题（主题色渐变底纹+左边框）> 日期（主题色浅底纹）> 标签（灰色系）
- 标签自动渲染为不同灰色，`#tag` 拆出独立展示
- 笔记间用渐变线 + 居中菱形 `◆` 分隔
- 弹窗顶部提供"复制全文"按钮
- 支持 Markdown 渲染为列表式排版

## 二、星点里程碑系统

废除独立能力值计算（采集力/输出力），用户成长通过**累积星点**直接体现——获得星点即获得成长，无需额外抽象层。

### 2.1 星点档位与增益

累积星点达到阈值后自动解锁对应效果，增益固定不累加（每档独立生效）：

| 累积星点 | 增益效果 |
|---------|---------|
| ≥ 500 | 解锁称号「初露锋芒」🗡️ |
| ≥ 1000 | 第 4 条采集倍数从 0.2 提升至 0.4 |
| ≥ 2000 | 内容输出奖励下限提升至 850 星点 |
| ≥ 3000 | 基金加成额外 +5% |

解锁触发由勋章系统统一管理（见第八章）。前端能力面板改为展示累计星点 + 当前增益档位。

### 2.2 连接力（保留独立计算）

连接力保留独立计算逻辑（标签共现图密度、节点数、跨界占比），但不再作为前端展示的"能力值"之一。连接力用于后台触发一次性星点奖励：

| 连接力达到 | 奖励星点 | 说明 |
|-----------|---------|------|
| ≥ 5 | +50 | 入门激励 |
| ≥ 20 | +100 | 深度连接 |
| ≥ 50 | +200 | 大师级 |

每个阈值只触发一次，奖励星点计入累积星点档位。

**连接力详细规则**（保留不变）：

- **设计意图**：引导用户从发散采集转向收敛思考。奖励密度而非广度——在已有标签之间反复缝合，而非不断开新标签。
- **评价逻辑**：标签两两配对，共现次数 +1；单个标签使用频次与连接力无关；连接力的增长引擎不是"新标签"，而是"已有标签之间的新连接"。
- **更新规则**：灵感提交时，对截断后的标签（前 5 个）做两两不重复组合，每条边权重 +1。若使用 ≥2 个不同领域标签，跨界笔记计数 +1。
- **行为引导**：新增孤立标签 → 节点数+1但不产生边，连接力可能下降；已有标签之间新连接 → 边权重+1，直接推高图密度。
- **公正性保障**：标签由 AI 生成、用户粘贴。AI 不知道连接力规则，用户无法刻意操纵跨界组合。

### 2.3 连接力变更日志

系统记录连接力每次变化的原因，在仪表盘展示最近 7 天的变更摘要。将抽象的数值还原为具体的行为故事——例如"连接力 +0.5，因为首次连接 `#冥想` 和 `#生产力`"。

### 2.4 赛季星点上限

每赛季星点上限 3000。赛季结算时软重置：保留 50% 星点进入下赛季，其余清零。避免星点无限膨胀，同时保留跨季连续性。

## 三、赛季系统

赛季是最高层周期（日→周→月→季），提供"荣誉退休"和"重新出发"的节奏，避免中断即弃坑。

### 3.1 赛季定义

- 默认长度：90 天（可配置），与自然季对齐
- 每个赛季有可选主题，引导用户探索不同行为模式
- 排行榜仅自己对比：本赛季 vs 上赛季同期

### 3.2 赛季主题

赛季主题系统（连线大师/深度矿工/分享者）为 v0.3 延展功能。当前版本所有赛季默认使用「开拓者」主题，无额外加成。开拓者勋章在赛季结算时正常颁发。

### 3.3 赛季结算

- 冻结当前赛季数据，存入历史
- 生成赛季结算报告（含"致未来自己"段落）
- 重置赛季计数器（赛季星点、活跃天数等）
- **不重置**：连续天数、能力值、基金池（终身累积）
- 颁发赛季专属勋章（永久）
- 下一赛季结算时引述上一季预测进行对比，建立连续性

## 四、标签宇宙

### 4.1 标签云与排行榜

- 数据源：`tag_graph.nodes`
- 字号映射：`log(count+1)` 决定大小，热门标签更突出
- 颜色深浅：基于共现连接度映射色阶
- 支持标签云和排行榜两种视图切换

### 4.2 标签点击交互

点击任一标签 → 弹窗展示：
1. 标签基础信息（使用次数、首次出现日期、关联 top5 标签）
2. 关联笔记列表（标题、日期、附带标签），按日期倒序
3. 笔记不展示正文，用户可前往 Obsidian 查看详情
4. 关联标签可跳转，形成探索流

## 五、兑换系统

### 5.1 路径选择：每周循环，不可跨周反悔

- 每个自然周为一个周期，用户选择基金或消费券
- 当周选择后不可更改，下周可重新选择
- 非永久绑定，而是持续性每周心理博弈

### 5.2 连续选择奖惩

| 连续选择 | 基金用户 | 消费券用户 |
|---------|---------|-----------|
| 连续 2 周 | 汇率 +0.05 | 汇率 -0.05 |
| 连续 4 周 | 汇率 +0.10 | 汇率 -0.10 |
| 连续 8 周 | 汇率 +0.15 | 汇率 -0.15 |
| 中断（换路径） | 汇率回到 1.5 | 汇率回到 1.0 |

基金基础汇率：1.5。消费券基础汇率：1.0。

### 5.3 兑换内容（具象化、可比较）

| 方向 | 兑换内容 | 用户获得 | 设计意图 |
|------|---------|---------|---------|
| 消费额度 | 星点按消费券汇率兑换 | 真实消费额度，可买衣物等 | 让你爽，然后让你后悔 |
| 基金投资 | 星点按基金汇率兑换 | 真实基金投资额度，可投指数基金 | 让你克制，然后让你骄傲 |

两个方向都是可晒的——消费晒实物，基金晒储蓄。但自律本身不可晒（晒了变质，变成自我感动）。

### 5.4 基金门槛

基金兑换需同时满足：
- **时间门槛**：首次开通后锁定 30 天
- **额度门槛**：基金池至少达到 500 星点

未满足时按钮置灰，hover 显示还差多少。

### 5.5 镜像对比（核心干预机制）

每次兑换确认页**强制展示**平行选择镜像，不判断、不说教，只展示事实：

**选消费时**：
- 等值对比：消费额度 vs 基金额度（含当前汇率）
- 时间维度延伸：基金未来增值预估 vs 消费品折旧
- 累计损失数字：本周/本月消费损失总额

**选基金时**：
- 等值对比：基金额度 vs 消费额度
- 累计克制收益：连续选择基金的增值累计

镜像让用户无法逃避自我审视。结合"造假羞愧感"——系统不抓你造假，但镜像让你无法对自己撒谎。

### 5.6 基金池

- `fund_pool` 与 `available_star` 分离存储
- 基金池内星点不可逆向转回消费券
- 仪表盘以动态水位图展示，星点增加有生长动画
- 能力值提升导致基金加成上涨时，有解锁提示

### 5.7 周期性选择回顾

**月度战报**：展示本月兑换选择总结，不加评判，让用户自己面对历史选择模式。

**赛季结算**：展示赛季兑换总览 + "这是你对自己的定义"，让用户判断"这个定义，我满意吗"。

## 六、连续加成（替代惩罚）

废除"连续中断收益减半"的惩罚机制。改为连续天数带来递增额外收益，中断后加成归零但基础收益不变。

| 连续天数 | 加成 |
|---------|------|
| 7 天 | 每日第 1 条采集额外 +10% |
| 30 天 | 每日第 1 条采集额外 +30% |
| 90 天 | 每日第 1 条采集额外 +50% |

中断时系统提示："你连续 X 天的采集记录中断了。但这 X 天已永久改变你的能力值。现在开始新的连续？"

**设计意图**：不是威胁（"不做就打你"），而是诱惑（"做了就有奖励，中断只是回到起点"）。压力来自内部（"我不想失去我已经有的东西"），而非外部（系统惩罚）。在长期主义的情境下，诱惑比威胁可持续得多。

## 七、三层反馈体系

覆盖从秒级到年级的整个等待期，让长期回报在当下可见。

### 7.1 微观反馈（采集瞬间）

每次采集成功，前端反馈三要素：
1. **行为确认**：笔记标题
2. **连接增量**：本次产生的新标签连接（如有）或最近活跃连接
3. **长期预告**："这条灵感将在本周日的周回顾中与你重逢"

同时返回基于标签重叠的相似笔记（最多 3 条），强化"与过去的自己对话"。

### 7.2 中观确认（能力值与周期回顾）

- 能力值面板 + 最近 7 天变更日志
- 周回顾、月战报中的一句话数据叙事（如"本周你的标签网络新增 3 个连接"）
- 周回顾中刻意突出峰值时刻（本周最佳连接、最长笔记、最跨界灵感）

### 7.3 宏观推演与预测

- **长期推演仪表**：基于当前速度，预测 6 个月后的笔记数、连接数、基金价值（纯数学计算）
- **赛季结算"致未来自己"**：赛季末生成预测，下赛季结算时对比回顾，建立连续性

## 八、勋章系统

所有勋章统一归入本章管理，不再区分来源。新增勋章只需在 `config.yaml` 的 `medals` 配置表添加条目，无需修改引擎代码。

### 8.1 勋章配置表（config.yaml）

```yaml
medals:
  - id: "first_triple"
    name: "灵光乍现"
    icon: "💡"
    trigger: "event"
    event: "daily_capture_count"
    condition: "count >= 3"
    once: true

  - id: "weekly_hunter"
    name: "周常猎人"
    icon: "🏹"
    trigger: "event"
    event: "weekly_review_streak"
    condition: "streak >= 4"

  - id: "link_master"
    name: "连线大师"
    icon: "🕸️"
    trigger: "event"
    event: "monthly_report_cross_domain"
    condition: "count >= 3"
    once: true

  - id: "hunt_apprentice"
    name: "采集达人"
    icon: "📚"
    trigger: "ability"
    ability: "hunt_power"
    threshold: 30

  - id: "link_novice"
    name: "连线新手"
    icon: "🔗"
    trigger: "ability"
    ability: "link_power"
    threshold: 5

  - id: "season_pioneer"
    name: "开拓者勋章"
    icon: "🗺️"
    trigger: "season"
    season_theme: "pioneer"
```

### 8.2 勋章触发类型

| 触发类型 | 说明 |
|---------|------|
| `event` | 特定事件触发（如单日采集3条、连续4周完成回顾） |
| `ability` | 能力值达到阈值时触发 |
| `season` | 完成赛季主题目标时触发 |

### 8.3 勋章检查机制

引擎实现 `check_medals()` 函数，在以下时机检查并发放勋章：
- 每次采集成功后（event 类型）
- 能力值更新后（ability 类型）
- 赛季结算时（season 类型）

`once: true` 的勋章只能获得一次，不可重复发放。

## 十、兑换确认页交互规范

用户点击兑换后，展示确认弹窗，必须包含以下内容（不可跳过）：

1. **等值对比**：本次兑换金额 vs 另一路径金额
2. **时间维度延伸**：基金未来增值预估 / 消费品折旧
3. **历史累计**：本周/本月消费损失，或累计克制收益
4. **两个按钮**："坚持兑换" 和 "我再想想"，后者视觉更醒目

## 十一、三层配置体系

系统使用三层配置文件，分离静态模板、运行时配置和用户数据：

| 文件 | 定位 | 修改方式 |
|------|------|---------|
| `defaults.yaml` | 静态模板：首次启动或重置时的初始值 | 手动编辑 |
| `config.yaml` | 运行时可调：倍率、阈值、汇率、解锁条件 | 编辑或 `/api/config` |
| `state.json` | 运行时累积：用户实际数据 | 系统自动更新 |

### 11.1 defaults.yaml（初始值模板）

`data/inspire/_狩猎系统/defaults.yaml` 定义首次启动或重置时的初始值模板。迁移脚本和重置接口从这里读取默认值，不在代码中硬编码。

### 11.2 config.yaml（运行时配置）

`data/inspire/_狩猎系统/config.yaml` 定义运行时可调整的参数，包括：
- 采集倍率（base_star, daily_multipliers）
- 任务奖励（weekly_review_reward, monthly_report_reward 等）
- 连续加成（streak_bonuses）
- 基金参数（fund.base_rate, fund.lock_days 等）
- 路径选择奖惩（path_bonuses）
- 星点里程碑（star_milestones）
- 连接力奖励（link_power_rewards）
- 跨界采集加成（cross_domain_bonus）
- 赛季星点限制（season_star_cap, season_soft_reset_ratio）
- 勋章配置（medals）
- 赛季配置（seasons）

### 11.3 state.json（用户数据）
